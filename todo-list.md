# 项目任务清单与实际进度

状态说明：`[x]` 已完成当前范围；`[~]` 已有可运行实现但仍缺少完整能力或验收；`[ ]` 尚未实现。

审计基线：2026-07-15。已验证 `npm run check`、`npm run build` 通过；`npm run lint` 当前因 `electron/main/index.ts:246` 的 `_history` 未使用错误失败，另有 3 个 React Hook 警告。项目暂无自动化测试文件和 `npm test` 脚本。

## 1. 工作区与路由

- `[~]` 工作区布局：左侧 API 树、顶部协议导航、环境选择器、请求标签页和保存状态已联动；目录展开状态仍是组件内状态，未持久化。
- `[x]` HTTP、WebSocket、Socket、环境、历史和设置路由已建立，协议页面可以切换。
- `[~]` 请求标签页支持打开、切换、关闭、关闭其他和关闭全部，标签顺序可拖拽并保存到偏好；尚未形成独立的通用标签页抽象。
- `[~]` 根路径跳转到 HTTP；加载、空态和部分错误提示存在，但 Home/Empty 仍不是独立完整首页体验。

## 2. API 目录管理

- `[~]` 目录树支持展开/收起、选中、搜索过滤和拖拽移动；展开状态未持久化，搜索只按名称匹配。
- `[x]` 支持新建目录、重命名、删除目录及其树节点。
- `[x]` 支持新建 API、编辑请求、重命名、删除和移动到目录。
- `[~]` 删除目录只移除目录树节点，关联请求定义可能保留为孤立数据，尚未有清理确认和迁移策略。
- `[x]` 目录菜单的新增子目录、新增 API、重命名、删除和移动入口已接入。

## 3. 新建与导入

- `[x]` 新建 API 弹窗：名称、目录、协议和 HTTP 方法可配置并保存。
- `[~]` 新建目录弹窗：名称和父目录可配置；排序、可见性和预览尚未实现。
- `[x]` 支持从常见 curl 命令解析 URL、方法、Headers 和 Body，并创建 HTTP API。
- `[ ]` Postman 导入。
- `[ ]` OpenAPI JSON/YAML 导入。
- `[ ]` 独立 URL 导入、批量合并策略和导入预览。

## 4. HTTP 请求调试

- `[x]` 方法、URL、Params、Headers、Body 和 Bearer Token 为受控编辑状态，支持 JSON 格式化、常用 Header 快捷填充和环境变量替换。
- `[~]` 认证目前主要支持 Bearer Token；Basic Auth、API Key、请求级变量和更完整的缺失变量校验仍待补齐。
- `[x]` Electron 主进程 HTTP 请求链路已接通，支持请求体、超时、响应头、响应体、状态码、耗时和大小统计。
- `[~]` SSE 响应可通过 HTTP 流实时读取并展示流式/原始内容；尚无独立 SSE 页面、停止/重连控制和指标面板。
- `[~]` 响应 Body、Headers、Cookies、日志标签已存在；Cookies 当前为空态，日志为基础状态文本，尚无完整响应断言执行。
- `[x]` 请求保存、工作区持久化、响应复制和 HTTP 请求历史写入已接入。
- `[~]` 网络、超时、无效请求和 IPC 错误已有结构化提示；非 2xx 语义、DNS/TLS 分类、重试和排查建议尚未完整归一化。

## 当前风险

- 没有自动化测试，现有检查只能证明类型检查和生产构建基本可用。
- lint 当前存在 1 个错误和 3 个警告。
- secret 仍会以明文写入工作区 JSON，UI 掩码不等同于安全存储。
- 工作区读取异常可能回退默认数据，权限错误或数据损坏不易被发现。
- WebSocket 使用 renderer 原生连接，HTTP/SSE/Socket 使用 Electron 主进程，协议能力和错误模型尚未统一。
